## Карта модулей v0.1 (детали — в .docx)
1. **О системе** — единая веб-панель, питает сайт и Mini App, одна БД. Не бухгалтерия.
2. **Структура** — Дашборд, Каталог, Склад, Заказы, Клиенты, Скидки/акции, Контент, Выгрузки/отчёты, Настройки.
3. **Роли и доступ** — логин/пароль (отдельно от TG), матрица Владелец/Менеджер/Кладовщик.
4. **Дашборд** — новые заказы, выручка, заказы по статусам, заканчивающиеся SKU, быстрые действия.
5. **Каталог** — 5.1 список (фильтры/массовые действия), 5.2 карточка (основное/категории/цена/фото S3/описание/характеристики/СДЭК-габариты/SEO), 5.3 импорт-экспорт + CommerceML, 5.4 категории (дерево 2 уровня, канонич.+доп.разделы), 5.5 характеристики.
6. **Склад** — источник правды = журнал движений. Остатки, приёмка, списание/корректировка, инвентаризация, журнал, резервы.
7. **Заказы** — список, карточка (серверный пересчёт сумм, СДЭК, ЮКасса, история), статусы (Черновик→Новый→Оплачен→Собран→Отправлен→Доставлен→Завершён; Отменён/Возврат), ручное создание, возвраты, уведомления.
8. **Клиенты** — список, карточка (объединение TG+web по телефону/email, история, адреса, теги), сегменты.
9. **Скидки и акции** — промокоды (из Mini App), скидки на товары, правила акций, автораздел «Распродажа».
10. **Контент (CMS)** — баннеры, коллекции, вдохновение (лукбуки), страницы, блог, подарочные карты (на уточнение).
11. **Выгрузки/отчёты** — для бухгалтера (Excel/CSV + ссылка на чек), CommerceML (1С), отчёты (продажи/топ/остатки).
12. **Настройки** — реквизиты, СДЭК (отпр. СПб, Дубровская 13), ЮКасса+54-ФЗ+НДС, пользователи/роли, SEO-шаблоны, юрдокументы (152-ФЗ), уведомления.
13. **Сквозные** — глоб.поиск, аудит действий, адаптив, производительность, безопасность (серверный пересчёт, данные в РФ).
14. **На уточнение** — подарочные карты, авторизация сайта (SMS vs email), НДС/налоги, глубина фильтров, матрица прав, вариативность (реком. 1 SKU = 1 товар).

Оценки заказчика по v0.1: архитектура 10, процессы 9, масштабируемость 10, удобство 9; **управление продажами 6, CRM 6, аналитика 5, автоматизация маркетинга 4**. Вывод рецензента: «сильная ERP/учётная система, но не инструмент роста продаж».

## Блок v0.2 — Воронка продаж, CRM-маркетинг и аналитика (PENDING)

**СТАТУС:** предложено заказчиком (рецензия ТЗ + 6 сообщений Арины, 30.06.2026). **НЕ входит в 470k v1.** Требует согласования скоупа и отдельного бюджета/фазы. Рецензент сам называет это «v0.2 / следующий шаг» — использовать как аргумент при разговоре о деньгах.

### Три фундамента (построив их — закрываем большинство запросов)
- **F1. Слой событий (analytics events)** — таблица событий + инструментирование витрины и Mini App. Фундамент для: воронки карточки, трекинга поиска, брошенных корзин, таймлайна клиента. Сейчас этих данных не собирается вообще.
- **F2. Клиент 360 + производные метрики** — агрегаты по существующей `orders` (дёшево) + сегментация.
- **F3. Слой сообщений/автоматизации** — email + TG-рассылки, сценарии. Самый тяжёлый, самый «продающий».

### Запросы Арины и триаж
**Дёшево (агрегаты по `orders`, кандидат в ранний goodwill / v1.1):**
- LTV в карточке клиента: сумма покупок, кол-во заказов, средний чек, последняя покупка. *[Арина п.1]*

**Средне:**
- RFM-сегментация (VIP / постоянные / новые / спящие / потерянные): пороги + джоб + UI. Запуск рассылок = зависит от F3. *[Арина п.2]*
- «Покупают вместе» (market-basket по `order_items`) → рекомендации / комбо / допродажи. Нужен объём заказов для сигнала. *[Арина п.4]*

**Тяжело (нужна новая инфраструктура):**
- Воронка карточки товара: просмотры → в корзину → оформление → покупка. Нужен F1. *[Арина п.5]*
- Трекинг внутреннего поиска (что искали, но не нашли) → SEO + ассортимент. Нужен поиск на витрине + логирование. *[Арина п.6]*
- Брошенные корзины + кнопка «отправить ссылку / напоминание». Нужны: серверная персистентность корзины + контакт до оплаты + канал (F3). Сейчас корзина витрины клиентская (Zustand). *[Арина п.3]*

### Из рецензии дополнительно (в тот же этап)
- **Аналитика:** конверсия сайт / Mini App, повторные покупки, средний чек, CAC, ROMI, ABC / XYZ, продажи по категориям / коллекциям.
- **Таймлайн клиента:** подписался / добавил / бросил / открыл письмо / промокод / написал в TG / отзыв / визиты.
- **Сценарии автоматизации:** 24ч после брошенной → сообщение; 30 дней → новая коллекция; ДР; после покупки → отзыв; после 2-го заказа → VIP.
- **Dashboard руководителя (KPI):** план продаж / выполнение %, средний чек, новые клиенты, повторные, ROI рекламы, конверсия, остаток ДС, заказы в работе, проблемные.
- **Поставщики (модуль):** контакты, закупки, сроки, задолженность, история, рейтинг, средний срок поставки.
- **Финансовый блок:** себестоимость, валовая прибыль / по товару / категории / поставщику.
- **AI-помощник (перспектива):** описания, SEO-тексты, цена, предупреждения об остатках, прогноз спроса, подбор для акций.

Приоритеты рецензента: 1) CRM-маркетинг, 2) BI-панель руководителя, 3) задачи сотрудников, 4) закупки, 5) AI.
